# Accessibility (A11Y)

GrokCode targets **keyboard** and **screen-reader** basics (not a full WCAG audit).
Use this checklist for NVDA / VoiceOver walkthroughs.

## Built-in helpers (`renderer/a11y.js`)

| Helper | Role |
|--------|------|
| `announce(msg, { assertive, force })` | SR live regions (polite throttled; assertive for errors / interactions) |
| `trapFocus` / `releaseTrap` | Settings, help, Inbox focus cycle |
| `presentInteractive` | Plan / ask / permission bars: region + announce + focus first control + **roving keys** |
| `bindMenuKeyboard` | Model / effort menus |
| `bindRovingKeyboard` | Shared Arrow/Home/End (+ Left/Right) for option groups |
| Skip link | “跳到输入框” → `#prompt` |
| Landmarks | main, explorer, chat, conversation log, tablists |

Toasts (`utils.toast`) also call `announce` (errors assertive).

## Manual walkthrough (NVDA / VoiceOver)

### Core shell

1. **Boot** — skip link focuses composer (`#prompt`).
2. **Settings** — open settings; Tab cycles only inside modal; Esc / close restores focus.
3. **Send message** — status chip / toast announced; stream lands in conversation `role=log`.
4. **Model / effort menus** — chips have `aria-haspopup`; Esc closes menus;
   **Arrow Up/Down · Home/End** move focus among items.
5. **CLI mode chip** (`#sbMode`) — button role; Enter/Space cycles mode (when ACP warm).
6. **Reduced motion** — OS reduce-motion or Settings “force reduce motion” stops decorative FX.
7. **Settings save** — if warm ACP was recycled, toast announces the count.

### Plan / ask / permission (CLI reverse-reqs)

8. **Plan approval** — bar announced assertively; focus on primary approve.
   - **Arrow Left/Right or Up/Down** among Approve YOLO / Ask tools / Revise / Quit.
   - Home / End jump to first / last action.
9. **User questions** — questionnaire announced; radio/checkbox + action buttons operable by keyboard.
10. **Tool permission (careful / non-YOLO)** — region labeled with tool title.
    - Options expose **CLI optionId** in `aria-label`.
    - Compact file-write row: same roving keys (Left/Right natural).
    - Cancel has explicit “取消授权 / Cancel permission” name.
11. **Standing grant** — subsequent auto-allow may only toast/Live status (no bar); listen for “standing grant”.

### Inbox

12. **Open Inbox** — `Ctrl+Shift+I` or topbar ◎; dialog traps focus; Esc closes.
13. **Live items** — list/listitem; act with buttons (plan tiers, permission options).
14. **Stale items** (after restart) — status banner: cannot reply; only Dismiss / Clear stale.
15. **Clear stale** — toolbar when any stale; command palette “清除 Inbox 过期项”; leaves live parks.
16. **Badge** — muted when only stale remain.

### Help cheatsheet

17. **?** / `Ctrl+/` — dialog + focus trap; lists Inbox shortcut.

### Themes (high contrast)

18. **Settings → Theme** — **高对比 · 黑底** (`hc`) or **高对比 · 白底** (`hc-light`).
19. Decorative nebula/starfield disabled; glass becomes solid; focus rings 3px.
20. Command palette “切换下一主题” cycles through packs including HC; direct **高对比** commands.
21. **`prefers-contrast: more` / `forced-colors`** — once-ever soft toast suggesting HC (never auto-applies).

### Diff keyboard

22. On **Diff** tab (not typing): **j/k** or **↑↓** files; **n/p** hunks; **a** review; **s** side-by-side; **[/]** scrub turns.
23. **Shift+j/k** (or Shift+↑↓) — navigate and multi-select; **x** toggle current; **Esc** clears multi-select.
24. File/hunk/selection changes are announced via live region.

## Gaps (known)

- Full virtualized Live list may not expose every row to SR.
- Diff hunk navigation depends on `.diff-hunk-head` being present in the pane.
- Permission option group is not a full `role=radiogroup` (buttons stay discrete CLI options).

## How to report

File an issue with: OS, SR (NVDA/JAWS/VoiceOver), steps, expected vs actual.
Label: `a11y`.
