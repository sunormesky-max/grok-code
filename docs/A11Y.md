# Accessibility (A11Y)

GrokCode targets **keyboard** and **screen-reader** basics (not a full WCAG audit).
Use this checklist for NVDA / VoiceOver walkthroughs.

## Built-in helpers (`renderer/a11y.js`)

| Helper | Role |
|--------|------|
| `announce(msg, { assertive, force })` | SR live regions (polite throttled; assertive for errors / interactions) |
| `trapFocus` / `releaseTrap` | Settings and modal focus cycle |
| `presentInteractive` | Plan approval / ask_user bars: region + announce + focus first control |
| Skip link | “跳到输入框” → `#prompt` |
| Landmarks | main, explorer, chat, conversation log, tablists |

Toasts (`utils.toast`) also call `announce` (errors assertive).

## Manual walkthrough (NVDA / VoiceOver)

1. **Boot** — skip link focuses composer (`#prompt`).
2. **Settings** — open settings; Tab cycles only inside modal; Esc / close restores focus.
3. **Send message** — status chip / toast announced; stream lands in conversation `role=log`.
4. **Plan approval** — when agent exits plan mode, bar is announced assertively; focus on Approve.
5. **User questions** — questionnaire announced; radio/checkbox operable by keyboard.
6. **Model / effort menus** — chips have `aria-haspopup`; Esc closes menus;
   **Arrow Up/Down · Home/End** move focus among items.
7. **CLI mode chip** (`#sbMode`) — button role; Enter/Space cycles mode (when ACP warm).
8. **Reduced motion** — OS reduce-motion or Settings “force reduce motion” stops decorative FX.
9. **Settings save** — if warm ACP was recycled, toast announces the count.

## Gaps (known)

- Full virtualized Live list may not expose every row to SR.
- Diff viewer keyboard depth is limited.
- No dedicated high-contrast theme yet (theme packs only).

## How to report

File an issue with: OS, SR (NVDA/JAWS/VoiceOver), steps, expected vs actual.
Label: `a11y`.
